# План API

Тонкий read-only API поверх схемы `main`. Каждый отчёт из `reports.sql` →
один эндпоинт. Стек — NestJS + PostgreSQL.

## Эндпоинты

| Метод | Путь | Назначение | Параметры |
|-------|------|------------|-----------|
| GET | `/funnel` | воронка: кол-во и сумма по стадиям | `from`, `to` |
| GET | `/managers` | продажи и маржа по менеджерам | `from`, `to`, `won` (только успешные) |
| GET | `/receivables` | дебиторка: сделка / оплаты / остаток | `min_remaining` |
| GET | `/production/delays` | задержки производства | `days` (по умолч. 5) |
| GET | `/deals/stale` | сделки без активности | `days` (по умолч. 7) |
| GET | `/sources` | выручка/маржа/ROMI по источникам | `from`, `to` |
| GET | `/data-quality` | сводка проблем данных (`rejects`) | `severity` |

Формат ответа — JSON `{ data: [...], meta: { ... } }`.

## Архитектура

В `docker-compose` к существующему `postgres` добавляются три контейнера:
`nginx`, `view` (фронт, рендерит отчёты/дашборды) и `api` (NestJS). Всё за одним
доменом — наружу торчит только `nginx`, который проксирует маршруты на нужный
контейнер. Браузер видит один origin, поэтому CORS не нужен; TLS терминируется
на nginx.

```
Браузер
   │ https://dashboard.example   (один домен)
   ▼
┌──────────────── nginx (reverse proxy) ────────────────┐
│   /        → view                                      │
│   /api/*   → api                                       │
└──────┬───────────────────────────┬────────────────────┘
       │ статика/SPA                │ HTTP/JSON
       ▼                            ▼
┌──────────────┐            ┌──────────────┐   SQL   ┌───────────────────────┐
│ контейнер view│            │ контейнер api│ ──────► │ контейнер postgres    │
│ (фронт)       │            │ (NestJS)     │         │ схема main + вьюхи v_* │
└──────────────┘            └──────────────┘         └───────────────────────┘
                     одна docker-compose сеть
```

- **view** — отдаёт UI с отчётами; за данными ходит на `/api/*` (тот же домен).
- **api** — read-only REST поверх `main`; в БД ходит по сети (`postgres:5432`).
- **nginx** — единая точка входа: `/` → view, `/api/*` → api; один домен/порт,
  TLS, отсутствие CORS.

Поток запроса: браузер → `GET /api/funnel` → nginx → `api` →
`Controller → Service → Repository` → `SELECT * FROM main.v_funnel` → JSON →
view рендерит. Слои `Controller/Service/Repository` — структура кода **внутри
одного процесса** `api`, а не отдельные контейнеры.
