**какие инструменты использовали;**
Claude Code в терминале bash



**какие задачи отдавали AI**
1) Оценку задания и разбор на детали.
2) Оценку качества данных из таблиц.
3) Оценку по выбору платформы для запуска базы(Выбрал развернуть докер контейнер PostgreSQL с выгрузкой SQL для supabase)
4) Заполнение DATA_QUALITY.md данными из оценки качества(Для дальнейшей проверки и редактирования в процессе)
5) Создание схемы и наполнение raw таблицы.
6) Генерация конечной схемы - базово.
7) Генерация sql нормализации
8) Генерация sql отчетов и проверка.
9) Вспомогательные функции нормализации: парсинг разных форматов дат (parse_ts), транслитерация источника в латиницу для ключа (translit/source_key), журнал отбраковки rejects + функция log_reject.
10) Ведение логов взаимодействия в ai-log/ по шаблону.


**что проверяли вручную**
1) Структуру таблиц - базово(Тут анализировал что вынести в реляционную связь(Источник, валюта и тп))
2) Терминологию FSM/BPM и прочитал отличия и как лучше представить в UML
3) Выбор версии PostgreSQL для совместимости с supabase(16)
4) Проверка типов. Оптимизировал где уместно(Enum, smallint)
5) Проверка sql нормализации
6) Правила по которым отбрасываем товары/сделки. Тут старался максимально сохранить данные.
7) Сверка цифр отчётов




**где AI ошибался или мог ошибиться**
1) AI мог создать лишние ненужные абстракции
2) В формировании sql кода при размытом промте.
3) parse_ts: наивные даты без смещения (14.06.2026) трактуются в TZ сессии (контейнер UTC), а не как +05:00 — возможен сдвиг времени.



**какие решения приняли сами**
1) Выбор платформы на чем развернем postgresql и способ создания raw слоя из csv
2) Вынос источника в отдельную таблицу.
3) Корректировка типов данных, создание enum для ролей.
4) Дубли товаров: отдавать приоритет тем кто есть в сделках. Можно было еще слой дописать по дате если оба отсутствуют в сделках.
5) Обнулил связь в контактах на организацию C999(не существует)
6) Неизвестные стадии сделок вынес в отдельный статус UNKNOWN
7) Обнулил товар с отрицательной ценой. По сути возврата нет и такой суммы не может быть. Но делать положительным тоже не стоит.
8) Отбросил оплату без сделки.
