# 07 — SQL-отчёты (reports.sql)

## Промпт
> реализуй Сделать SQL-отчеты: текущая воронка (кол-во и сумма по стадиям); продажи и маржа по менеджерам; дебиторка (сумма сделки, оплаты, остаток); сделки с задержкой производства >5 дней; сделки без активности последние N дней; источники заявок и выручка/маржа; список проблем данных. В файле reports.sql.
> Затем: прогони SQL-запросы из reports.sql и выпиши короткий итог. задокументируй

## Ответ ИИ
> Написаны 7 отчётов в `reports.sql`, прогнаны на БД `crm` (контейнер `postgres`) без ошибок.

### Заложенные соглашения (в шапке reports.sql)
- Выручка по позициям: `quantity*unit_price - discount`; себестоимость: `quantity*products.cost_price`; маржа = выручка − себестоимость. Экономика только по `deal_products` (сделки D1001–D1005).
- «Оплачено» в дебиторке = `payments.status='paid'` (pending не зачитывается, показан отдельно).
- Точка отсчёта дат — `current_date` (в БД = 2026-06-24). N в отчёте об активности = 7 дней (один `INTERVAL`).
- Продажи по менеджерам — по всем сделкам, без фильтра по стадии (объём данных мал).

### Короткий итог прогона
1. **Воронка** — 11 сделок, по 1 на стадию (WON — 2). Деньги: WON 361k, CONTRACT 174k, PRODUCTION 150k; UNKNOWN 1 сделка (54k).
2. **Менеджеры** — U10: 362k выручки / 124.7k маржи (34.4%); U11: 176k / 90.9k (51.6%, лучшая маржинальность); U12: 57k / 13.8k.
3. **Дебиторка** — суммарный остаток ≈663k. Топ: D1010 179k (с учётом возврата 5k), D1009 125k, D1001 90k. D1003 закрыта, D1005 переплата −8k.
4. **Задержка производства >5 дней** — PO001 (12 дн, в работе, просрочен), PO003 (6 дн, завершён поздно).
5. **Без активности (N=7)** — 8 открытых сделок; у 5 активностей не было, у 3 последняя >2 недель назад; финальные стадии исключены.
6. **Источники** — Avito net 121.6k (затраты 8.4k), Yandex Direct net 77.4k, Website 13.8k; phone/referral/telegram — заявки без выручки.
7. **Проблемы данных** — 14 записей в `rejects`: 5 error (orphan D9999/D1012/PR999), 9 warning (дедупы, обнуления, дефекты).

### Что проверено вручную
- Сверены revenue/margin по менеджерам и источникам с ручным пересчётом позиций (D1001–D1005) — сходится.
- Дебиторка: переплата D1005 и возврат D1010 отрабатывают как ожидалось.
- Задержки и пустые активности соответствуют данным при current_date=2026-06-24.

### Возможные доработки (не реализованы)
- Продажи по менеджерам/источникам можно ограничить `is_success` стадиями (реализованные продажи).
- «Сумма сделки» в дебиторке = `expected_amount`, а не сумма позиций — отсюда отрицательный остаток D1005.
